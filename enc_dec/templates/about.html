SECURE FILE STORAGE

Using Hybrid Cryptography





Aim: To secure user files through hybrid cryptography for cloud storage.

Approach:

  1. Upload user file.
  2. Dividing an uploaded file into N parts depending on the fixed block size.
  3. The sub-files are encrypted using one of the selected algorithms that rotate implementation with every part in round robin fashion.
  4. The keys for cryptography algorithms used are stored in a file and then the resulting file is secured using a different cryptographic algorithm. Key is available to the user as public key, for the receiver to be able to access the file.

File Restoration:


  1. Load the public key (.pem file) from sender.
  2. Validate and decrypt keys of the algorithms.
  3. Decrypt blocks of subfiles using the same algorithms used for encryption.
  4. Combine all subfiles to form the original file and allow user download.
